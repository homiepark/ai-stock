> "철도 = AI" 사이클 가설을 구체적 종목으로 바꾼 결과. 19세기 철도 시대 4대 부의 원천(땅·강철·자금중개자·운영자)을 AI에 대입.

## 🎯 큰 그림: 5개 테마

|---|---|---|---|---:|
| 1 | 전력·그리드 | "땅"(부동산) | AI의 진짜 병목은 GPU가 아니라 **전력**. 미국 변압기 리드타임 100주+. | 13 |
| 2 | 반도체·HBM·패키징 | "강철"(카네기) | HBM4→HBM5 사이클 2027까지 가시성. CoWoS 캐파 부족 지속. | 15 |
| 3 | 하이퍼스케일러 | "자금중개자"(JP모건) | Capex 사이클의 자금줄. 2026년 Capex 합계 4,000억 달러+. | 7 |

| 5 | Embodied AI/로보틱스 | (다음 게임체인저) | 인지노동 다음은 **물리노동의 종말**. | 5 |

총 48종목 (미국 36 / 한국 12 = 75% / 25%).

## 🏷️ 등급 (tier) 분류

각 테마 안에서 종목은 3가지 등급:

| 등급 | 기호 | 의미 | 예시 |
|---|---|---|---|
| **leader** | ★ | 시총·점유율 1~2위 대장 | NVDA, GEV, MSFT |
| **momentum** | ▲ | 모멘텀 폭발 가능성 큰 변동성 종목 | OKLO(SMR), ALAB, APP |
| **supporting** | · | 사이클의 후순위 수혜주 | ETN, MU, AMAT |

**왜 등급을 나눴나**: 동일 테마라도 진입 전략이 다름. Leader는 분할매수로 길게 보유, Momentum은 변동성을 인식하고 비중 작게.

## 🔍 종목 선별의 기본 원칙 (적용된 필터)

이 50종목 모두가 다음 5가지 필터를 통과:

1. **AI 사이클의 직접 수혜** — "AI 들먹이는" 회사가 아니라 "AI Capex 사이클이 매출로 직접 변환되는" 회사만.
2. **공급 제한 또는 시장 지배** — 가격 결정력이 있는 회사. 동일 카테고리에서 중복 가능한 회사는 1~2개로 압축.
3. **시총·유동성 충분** — 미국 시총 10억 달러+, 한국 시총 5천억 원+. 거래량 부족한 마이크로캡 제외.
4. **재무 정보 공개** — yfinance/네이버금융으로 펀더멘털 정량 분석 가능한 회사. SPAC·신생 IPO 일부 제외.
5. **테마 분산** — 한 테마에 몰빵하지 않도록 5개 테마에 균형.

## ✏️ 워치리스트 직접 편집

`config/universe.yaml`을 직접 수정하면 됩니다:

```yaml
themes:
  vertical_ai:
    name: "버티컬 AI / 운영자"
    thesis: "..."
    stocks:
      - { ticker: ADBE, country: US, tier: supporting, name: "Adobe", note: "AI 위협 회복 트래킹" }
      # 새 종목 한 줄 추가
```

저장 후 `uv run ai-stock universe`로 확인 → `uv run ai-stock daily`로 다음 리포트부터 반영.

> **Tip**: 일회성 호기심은 워치리스트 편집보다 `ai-stock analyze TICKER`가 빠릅니다 (다음 섹션 참조).

## 🔎 워치리스트 외 종목 검색

```bash
ai-stock analyze NVDA              # 미국 주식
ai-stock analyze 005930            # 한국 주식 (6자리 코드)
ai-stock analyze AAPL --name 애플   # 이름 지정
```

`reports/adhoc/<TICKER>-YYYY-MM-DD.md`에 저장. 워치리스트와 동일한 단·중·장 점수 + Claude 서술 + 차트 스냅샷이 한 페이지로.

> 분석 결과를 보고 마음에 들면 → `config/universe.yaml`에 추가해 매일 추적하면 됨.
